# Finch Comparison & iOS Mobile Workarounds

- **Clarified core product differentiation from Finch:** The self-care pet is fundamentally different from Finch because it lives as a **roaming desktop companion** that reacts *in real-time* to your screen behavior (push/interrupt), rather than a mobile-app-based reward system for completed self-care actions (pull). Finch can't replicate the desktop roaming mechanic.

- **Identified key positioning constraint:** In short social clips (TikTok, etc.), the product can appear derivative ("cute pet + productivity"), so the entire marketing strategy must lean on the single most demonstrable differentiator—the visual of a pet physically running across your laptop during active scrolling.

- **Explored viable iOS mobile strategy:** Since iOS blocks roaming graphics, mapped out a constrained but on-brand iOS path using **communication notifications** (pet texts you with custom avatar), **lock-screen/home-screen widgets** (pet visible without opening app), and optional **Dynamic Island + custom Screen Time block screens**. Path A (schedule-based nudges, no Apple entitlement) is faster for MVP; Path B (screen-time-triggered, requires FamilyControls approval) is smarter.

- **Documented iOS tradeoffs:** Notifications are muteable and not real-time, widgets have refresh budgets and no animation, and Screen Time triggers are unreliable. The iOS version will feel different—glanceable and message-based—but is still shippable if built intentionally.

- **Captured findings in venture note:** iOS workarounds and Finch comparison analysis documented in the productivity pet venture file for positioning continuity.
